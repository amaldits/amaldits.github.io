Unmanned Aerial Vehicle

When I was in college working on my final thesis, I worked on a project related to UAVs (Unmanned Aerial Vehicles. The mission of this vehicle is to carry out inspections on bridges with as little human influence as possible (semi-autonomous).


Previously, the bridge must have a sensor that provides position, and status. So, if the bridge having trouble, the sensor will send a signal to GCS (Ground Control Station). The drone will receive the signal. If the condition is possible to do an inspection, the drone will take off with operator validation. The drone flies with a pattern depending on what situation the bridge has and takes video or image data for the surveyor or whoever wants to use the data. If the mission is done, the drone will fly to GCS.

Autonomous Tram


This project is a collaboration between ITB (Institut Teknologi Bandung), INKA (Industri Kereta Api), riset.ai, and LPDP (Lembaga Pengelola Dana Pendidikan) which aims to create a tram that can run autonomously. To assist it in running autonomously, the Tram has 9 cameras, a LIDAR, 2 RADAR, a GNSS, and an IMU. All the sensors communicate with the tram’s NVIDIA DRIVE AGX embedded computer which interfaces with the tram’s PLC. In this project, I was responsible for integrating the tram’s hardware system from power distribution to how the sensor data can be used to be processed after.

Autonomous Forklift

This project is a collaboration between ITB (Institut Teknologi Bandung), and CPIN (Charoen Pokphand Indonesia Tbk) which aims to create a forklift that can run autonomously, and locate itself in the indoor facility. The forklift has a LIDAR, and an IMU, which communicate to a mini PC.


My role in creating an autonomous forklift is to integrate all the sensors and serve the data so the data can be used on all the methods.

2 sensors generate data. The data that needs to be filtered. We filter using Kalman Filter. Those data are needed for implementing HectorSLAM so the forklift can map the warehouse and locate itself, so it can automate to carry a load, sign the destination, and drive to the location based on the map before, taking the load on its location, and back to home point.

Worker Tracker

Worker Tracker is a supporting device for tracking field workers' footprints in oil palm fields. The data can be visualized by a web-based application. We can check the region that must be completed on that day from the footprint data. Then, stakeholders or users can make decisions about what should be done by tomorrow, so this app can help users minimize the error, and increase certainty about the decision itself.



Worker Tracker is a wearable device. It is placed at human hips with a small compartment designed just for Worker Tracker. It doesn’t have any input interface, the interface includes an LED, which shows the status Worker Tracker has, and a USB Type-C, which charges the Worker Tracker and sends the data through Wi-Fi to the nearest Access Point at once.

In this project, I was working on the firmware side, which involved creating and designing the firmware, trial, and error the new features assigned by the project manager, and debugging the code if there were some bugs that needed to be fixed.

Humanoid Robo Soccer

This project is intended to take part in a competition, Kontes Robot Sepak Bola - Humanoid (KRSBI-H). The robot will walk, kick, and coordinate with another robot making a pass. The humanoid robot has 20 servos and every servo communicates with each other using RS485/TTL. All servos are assembled and connected electrically to a sub-controller, CM730. This sub-controller is used for “translate” the high-level code so we can generate a move.


This robot also has a camera and mini PC. Mini PC as a main controller generates commands based on what it detects an object with the camera such as a ball, goalpost, straight line, and round circle. In this project, I was working on maintaining the electrical part and creating the electrical part to support the robot to make it easier to use. I created an interface to simplify creating strategy before the game starts.
